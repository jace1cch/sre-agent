# 服务器部署说明

这份文档说明如何把当前 MVP 运维 Agent 部署到腾讯云 Linux CVM 宿主机上。

## 推荐目录

- 代码目录: `/opt/sre-agent`
- 环境变量文件: `/etc/sre-agent.env`
- 事故落盘文件: `/opt/sre-agent/data/incidents.jsonl`

## 部署步骤

### 1. 上传代码

把当前仓库部署到服务器，例如：

```bash
mkdir -p /opt/sre-agent
cd /opt/sre-agent
git clone <your-repo> .
```

### 2. 准备环境变量

推荐直接使用针对腾讯云 2 核 2G 单机环境的模板：

```bash
cp /opt/sre-agent/deploy/examples/tencent-cloud-cvm-2c2g.env /etc/sre-agent.env
```

如果你想从通用模板开始，也可以复制 `.env.example` 到系统环境文件：

```bash
cp /opt/sre-agent/.env.example /etc/sre-agent.env
```

然后按实际环境修改以下关键项：

- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`，默认可直接使用 `https://api.deepseek.com`
- `APP_CONTAINER_NAME`
- `APP_CONTAINER_NAMES`，多个容器用逗号分隔
- `WEBHOOK_URL`
- `REPOSITORY_PATH`
- `LOG_CLEAN_PATHS`

### 3. 验证单次诊断

```bash
cd /opt/sre-agent
PYTHONPATH=src python3 -m sre_agent.run
```

如果当前没有异常，输出 `No issues detected.` 属于正常现象。

### 4. 使用安装脚本

如果代码已经放在 `/opt/sre-agent`，可以直接执行：

```bash
cd /opt/sre-agent
bash deploy/install_server.sh
```

脚本会完成以下动作：

- 创建虚拟环境
- 安装当前项目
- 初始化 `/etc/sre-agent.env`
- 安装 `systemd` 服务文件
- 执行 `systemctl daemon-reload`

### 5. 安装 systemd 服务

```bash
cp /opt/sre-agent/deploy/systemd/sre-agent.service.example /etc/systemd/system/sre-agent.service
systemctl daemon-reload
systemctl enable sre-agent
systemctl start sre-agent
```

如果你已经执行了 `deploy/install_server.sh`，这一步可以直接跳过。

### 6. 查看运行状态

```bash
systemctl status sre-agent
journalctl -u sre-agent -f
```

## Docker 权限

当前 MVP 需要访问 Docker CLI，因此运行用户必须有 Docker 权限。最简单方式是以 `root` 运行，后续再收敛权限。

## Java 配合项

为了让业务层监控有效，Java 应用建议尽快补齐：

- 结构化业务日志，参考 `docs/业务日志格式规范.md`
- GC 日志输出
- 容器内可执行 `kill -3 1` 或 `jcmd` / `jstack`

## 建议上线顺序

建议先按下面顺序上线：

1. 先开检测与通知
2. 观察 1 到 2 天误报情况
3. 再开启 `AUTO_REMEDIATE=true`

这样更稳，不容易在第一天就因为阈值不准造成误操作。

## 快速自检

安装完成后，建议运行：

```bash
cd /opt/sre-agent
bash deploy/check_server.sh
```

这个脚本会检查：

- Python 是否可用
- Docker 是否可用
- `/etc/sre-agent.env` 是否存在
- `APP_CONTAINER_NAME` 或 `APP_CONTAINER_NAMES` 指向的容器是否可见
- 单次诊断是否能跑通

## Autonomous 部署前检查

建议在启动 `systemd` 服务前先运行：

```bash
cd /opt/sre-agent
PYTHONPATH=src .venv/bin/python -m sre_agent.cli.main check-deploy
```

如果你想拿到结构化结果，便于远端排查，可以运行：

```bash
cd /opt/sre-agent
PYTHONPATH=src .venv/bin/python -m sre_agent.cli.main check-deploy --json
```

重点看这几类输出：

- `linux_platform`: 必须通过
- `docker_binary` 和 `docker_daemon`: 必须通过
- `container_targets`: 最好通过，否则说明配置容器名不对或权限不足
- `prometheus`: 不通过不会阻塞启动，但指标输入会降级
- `codebase_path`: 不通过不会阻塞启动，但源码检索会降级
- `sources`: 会展示当前设计里哪些输入来源可用、缺失、或尚未接入
